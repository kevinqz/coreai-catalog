# v0.3 Verification Checklist

Core AI Catalog v0.3 moves from structural completeness toward verification depth.

## v0.3 goals

- Validate model records.
- Validate artifact records.
- Validate upstream taxonomy records.
- Validate benchmark records.
- Export JSON views for agent/API consumption.
- Track exact original upstreams separately from converted artifacts.
- Track review-needed source and license gaps explicitly.

## Source layers

| Layer | File | Status |
|---|---|---|
| Model facts | `catalog.yaml` | active |
| Converted artifact provenance | `artifacts.yaml` | active |
| Source taxonomy | `upstreams.yaml` | active |
| Benchmarks | `benchmarks.yaml` | active |
| Schemas | `schema/*.json` | active |
| Markdown views | `docs/*.md` | active |
| JSON exports | `dist/*.json` | generated |

## Review-needed records

These upstream records should be resolved with exact primary URLs before being marked high-confidence:

- GLM-4.7-Flash
- Nanbeige4.1-3B
- AdcSR ×4

- OpenRAIL-style license source

## License review

Sensitive or non-standard licenses should remain marked for review until source-specific terms are checked:

- Gemma Terms
- Meta SAM License
- LFM Open License
- OpenRAIL-style license family

## Done criteria for v0.3

- `python scripts/validate.py` passes.
- `python scripts/generate.py` runs (docs + JSON).
- README documents all source-of-truth layers.
- Review-needed upstreams are visible, not hidden.

## Next milestone: v0.4

v0.4 should focus on exact source depth:

- exact model-card URL for every original model
- exact license URL per model
- checksum/hash fields for artifacts when available
- benchmark prefill-latency, memory-footprint and QAT-variant rows
- generated JSON committed or published as release artifacts
