# SotA Maintenance Plan

This repository should stay compact, granular, source-grounded and validation-ready.

## Target architecture

```txt
catalog.yaml      = model facts
artifacts.yaml    = converted artifact provenance and download references
sources.yaml      = compact source registry
upstreams.yaml    = framework, original-model, benchmark, sample and license source taxonomy
benchmarks.yaml   = normalized benchmark records
schema/           = validation contracts
scripts/          = validators, doc generators and export tools
docs/             = generated or curated human views
dist/             = generated JSON exports
```

## Source-of-truth rule

YAML files are the source of truth. Markdown and JSON are derived views unless explicitly marked as curated.

## Generated views

- `docs/model-registry.md` is generated from `catalog.yaml`.
- `docs/artifact-provenance.md` is generated from `artifacts.yaml`.
- `dist/*.json` is generated by `scripts/export_json.py`.

## Curated views

These files are curated manually until generators are expanded:

- `docs/index.md`
- `docs/capability-matrix.md`
- `docs/runtime-matrix.md`
- `docs/upstream-map.md`
- `docs/benchmark-map.md`
- `docs/source-map.md`
- `docs/v0.3-verification.md`
- `docs/sota-maintenance.md`
- `docs/generated-files.md`

## Rules

1. One meaningful model variant = one catalog entry.
2. One converted downloadable artifact = one artifact entry.
3. Use `unknown` instead of guessing.
4. Keep license review explicit.
5. Keep model facts in `catalog.yaml`.
6. Keep converted artifact provenance in `artifacts.yaml`.
7. Keep original-model/framework/license/benchmark source taxonomy in `upstreams.yaml`.
8. Keep measurement rows in `benchmarks.yaml`.
9. Credit original model creator, conversion source and artifact host separately.
10. Regenerate derived files after editing YAML sources.

## Validation

```bash
python scripts/validate.py
python scripts/generate_docs.py
python scripts/generate_artifact_docs.py
python scripts/export_json.py
```

## Cross-reference checks

`validate.py` verifies:

- each `catalog.yaml` `artifact_ref` exists in `artifacts.yaml`
- each model source exists in `sources.yaml` or `upstreams.yaml`
- each benchmark `model_id` exists in `catalog.yaml`
- each benchmark `source` exists in `sources.yaml` or `upstreams.yaml`
- each `original_model_sources.applies_to` target points to a model or artifact ID

Conceptual framework/runtime `applies_to` values are allowed in other upstream groups.
