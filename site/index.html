Scoring

Each model gets a readiness score (0-100) based on 13 factors:

85-100 A Production-ready
70-84 B Good, minor gaps
55-69 C Usable, verify caveats
40-54 D Experimental
0-39 F Early / unverified
How scores are calculated
Artifact available+15
License: commercial likely+10
iPhone supported+10
Mac supported+10
Has benchmark+10
Stock runtime (no custom)+10
No custom kernel / patch / AOT+5 each
Status: confirmed+10
Confidence: high +5 / medium +3 / low -10variable
Maturity: stable or active+5

Contribute

Core AI Catalog is community-driven. Here's how to add models, submit benchmarks, report issues, and convert artifacts.

➕

Add a Model

Found a Core AI .aimodel artifact that's missing from the catalog? Use the YAML templates to add it:

  • Model entry template
  • Artifact entry template
  • Contribution guide (AGENTS.md)
bash
# Clone, add your entry to catalog.yaml + artifacts.yaml, then:
python scripts/validate.py
python scripts/generate.py
python scripts/audit.py
📊

Submit Benchmarks

Ran a benchmark on your Apple Silicon device? Append your measurement to benchmarks.yaml:

  • benchmarks.yaml
  • Benchmark JSON Schema

Include metric, value, unit, device, compute unit (GPU/ANE/CPU), and environment. Superseded values are retained with confidence: needs_review — never overwritten.

🐞

Report Issues

Spotted a data error, missing license, or broken provenance link? Open an issue:

  • Report a bug
  • Request a model
  • Submit benchmark data
  • All open issues
