# Contributing to Core AI Catalog

Thank you for helping build a more complete and trustworthy catalog of Apple Core AI models.

## Source of truth

YAML files are authoritative. Markdown docs and JSON exports are derived views generated by scripts. **Always edit YAML, never edit generated files directly.**

| Edit directly | Generated (don't edit) |
|---|---|
| `catalog.yaml` | `docs/model-registry.md` |
| `artifacts.yaml` | `docs/artifact-provenance.md` |
| `sources.yaml` | `docs/apple-terminology-map.md` |
| `upstreams.yaml` | `docs/index.md` |
| `benchmarks.yaml` | `dist/*.json` |
| `terms.yaml` | |
| `schema/*.json` | |

## Add a model

### 1. Add model metadata to `catalog.yaml`

```yaml
- id: my-model-1b
  name: MyModel-1B
  family: MyModel
  source_group: zoo          # zoo | official | external
  source_path: https://github.com/.../model-card.md
  artifact_ref: my-model-1b   # must match an id in artifacts.yaml
  capabilities:
    - chat
    - text-generation
  modalities:
    input:
      - text
    output:
      - text
  artifact:
    format: aimodel
    availability: available
  size:
    parameters: 1B
    precision: int8
    quantization: int8lin
    artifact_size: 850MB
  runtime:
    runtime_name: apple-core-ai
    runner: CoreAIRunner
    stock_runtime: false
    custom_kernel: false
    patch_required: true
    tokenizer_required: true
    processor_required: false
    aot_required: false
  device_support:
    iphone: true
    ipad: unknown
    mac: true
    mac_only: false
  license:
    name: Apache-2.0
    commercial_use: likely      # likely | check_license
  status: confirmed              # confirmed | needs_review | deprecated
  maturity: experimental         # stable | active | experimental | research
  confidence: medium             # high | medium | low | needs_review
  sources:
    - coreai-model-zoo-readme
    - coreai-model-zoo-zoo
  last_verified: '2026-06-30'
  notes: Optional notes about parity, caveats, etc.
```

### 2. Add artifact provenance to `artifacts.yaml`

```yaml
- id: my-model-1b
  group: zoo
  github:
    owner: converter-org
    repo: coreai-conversions
    path: https://github.com/converter-org/coreai-conversions/blob/main/zoo/my-model.md
  huggingface:
    owner: hf-user
    repo: MyModel-1B-CoreAI
    url: https://huggingface.co/hf-user/MyModel-1B-CoreAI
    # path: tree/main/vl       # optional sub-path within the HF repo
  officiality:
    apple_export_recipe: false
    apple_hosted_artifact: false
    community_packaged: true
```

### 3. Add source/upstream records (if missing)

If the converter, original model, or artifact host is not already in `sources.yaml` or `upstreams.yaml`, add it.

### 4. Add benchmark rows (if published)

```yaml
- id: my-model-1b-iphone17pro-gpu-toks
  model_id: my-model-1b
  metric: decode_throughput
  unit: tokens_per_second
  value: 52.3
  device: iPhone 17 Pro
  compute_unit: GPU
  environment: iOS 27 beta, coreai-pipelined engine
  observed: '2026-06-30'
  source: coreai-model-zoo-readme
  confidence: medium
  notes: Decode throughput from upstream README table.
```

## Required checks

Before submitting a pull request, run **all** of these and ensure zero errors:

```bash
python scripts/validate.py
python scripts/audit.py
python scripts/generate.py
```

Then verify no generated files are out of sync:

```bash
git diff --exit-code docs/ dist/
```

CI runs the same checks, so if they pass locally they will pass in CI.

## Officiality rules

- **`apple_export_recipe: true`** — use **only** for conversions that follow Apple's official export recipes from `apple/coreai-models`.
- **`apple_hosted_artifact: true`** — use **only** if Apple hosts the artifact themselves (currently always `false`).
- **`community_packaged: true`** — use for community-zoo packaged artifacts (e.g., `john-rocky/coreai-model-zoo` ports hosted on Hugging Face by `mlboydaisuke`).
- `source_group` must be consistent with `officiality`:
  - `source_group: official` → `apple_export_recipe: true`
  - `source_group: zoo` → `apple_export_recipe: false`

## Unknowns

Use `unknown` instead of guessing. The audit enforces zero `unknown` values in critical fields (`precision`, `quantization`, `artifact_size`, runtime flags). If the upstream doesn't disclose a value, use `not_published` instead of `unknown`.

## Submitting

1. Fork the repository.
2. Create a branch: `git checkout -b add-my-model`.
3. Make changes to YAML sources only.
4. Run all checks above.
5. Commit YAML changes + regenerated derived files.
6. Open a pull request describing what was added.
